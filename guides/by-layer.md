# 按架构层次导航

本导航按照系统的架构层次组织，从基础设施层到应用层，帮助你理解系统的整体架构。

## 🏗️ 架构层次概览

```
┌─────────────────────────────────────────┐
│         应用层 (Application)            │
│  Skills, Plugins, Remote Execution      │
├─────────────────────────────────────────┤
│        编排层 (Orchestration)           │
│  Task Framework, Agent, Scheduler       │
├─────────────────────────────────────────┤
│         核心层 (Core)                   │
│  Query Engine, State, Commands          │
├─────────────────────────────────────────┤
│        工具层 (Tools)                   │
│  Tool System, MCP, Permissions          │
├─────────────────────────────────────────┤
│        基础层 (Foundation)              │
│  Runtime, UI, Types, Session Storage    │
└─────────────────────────────────────────┘
```

## 📝 学习建议

### 推荐学习顺序

**第 1 周**: 基础层
- 00-overview → 01-runtime-foundation → 02-terminal-ui → 03-type-system

**第 2 周**: 工具层 + 核心层
- 04-tool-system → 06-query-engine → 07-state-management → 05-command-system

**第 3 周**: 编排层
- 08-task-framework → 10-cron-scheduler → 11-agent-coordination

**第 4 周**: 应用层 + 横切关注点
- 13-mcp-integration → 15-plugin-system → 16-skill-system → 其他

### 学习技巧

1. **自底向上**: 先理解底层机制，再学习上层抽象
2. **实践优先**: 每学完一层，尝试构建简单示例
3. **关注接口**: 理解层与层之间的接口设计
4. **绘制架构图**: 帮助记忆和理解层次关系
